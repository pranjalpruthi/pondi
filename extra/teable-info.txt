You can easily construct the command for any of the 399 API endpoints by following these patterns.

### General `curl` Command Structure

A typical `curl` command for the Teable API will look like this:

```sh
curl --request <HTTP_METHOD> \
  --url <API_ENDPOINT_URL> \
  --header 'Authorization: Bearer <your_personal_access_token>' \
  --header 'Content-Type: application/json' \
  --data '<JSON_BODY_IF_NEEDED>'
```

**Key Components:**

*   **`<HTTP_METHOD>`**: `GET`, `POST`, `PATCH`, `PUT`, or `DELETE`.
*   **`<API_ENDPOINT_URL>`**: The full URL for the API endpoint (e.g., `https://app.teable.io/api/table/{tableId}/record`).
*   **`<your_personal_access_token>`**: Your Teable Personal Access Token.
*   **`<JSON_BODY_IF_NEEDED>`**: The JSON data required for `POST`, `PATCH`, and `PUT` requests. `GET` and `DELETE` requests typically do not have a body.
